where/how to get structured data (e.g. JSON) for houses on sale? (e.g. Japan マンション)

You can obtain structured data (e.g., JSON) for houses on sale in Japan, including マンション (condominiums), primarily through official APIs and open-source tools that utilize data from Japan's Ministry of Land, Infrastructure, Transport and Tourism (MLIT). Here are the main methods and resources:

1. MLIT Real Estate Transaction Price Information API​

  • The Japanese government provides an official API for real estate transaction prices, including data on used condominiums, land prices, and other property types.
  • You need to apply for an API key via the official site.
  • The API allows querying by trade period, prefecture code, and municipality code to get detailed transaction data in JSON format.
  • Example data fields include price, floor plan, building year, structure, purpose, and location details.

How to use:​

  • Register for an API key.
  • Use HTTP requests with parameters such as year, area (prefecture code), and city (municipality code).
  • The response is JSON containing detailed real estate transaction records.

Example Python snippet:​

import requests

url = "https://www.reinfolib.mlit.go.jp/ex-api/external/XIT001"
headers = {"Ocp-Apim-Subscription-Key": "YOUR_API_KEY"}

params = {
    "year": 2023,
    "area": 13,  # Tokyo prefecture code
    "city": 13103  # Example municipality code for Minato-ku, Tokyo
}

response = requests.get(url, headers=headers, params=params)
data = response.json()
print(data["data"])  # List of transaction records in JSON

This API provides rich, official, and structured data suitable for research and analysis1.

2. Open-Source Python Wrappers and Tools​

  • j_realty_api: A Python wrapper for the MLIT API that simplifies fetching real estate transaction prices, supporting both Japanese and English.
  • It includes utilities to get prefecture and municipality codes.
  • Installation via pip and usage examples are provided on GitHub.
  • You can get data as pandas DataFrames or JSON easily.

Example usage:

from j_realty_api import j_realty_en as jrn

j = jrn.CityCode('Tokyo')
pref_code = j.pref_code
city_code = j.city_code('Minato')

pt = jrn.PropTransactions(pref_code, city_code, 20231, 20234)  # Q1 to Q4 2023
df = pt.get_data()
print(df.head())

This tool is convenient for developers and researchers working with Japanese real estate data23.

3. Web Scraping from Major Real Estate Portals​

  • For more real-time and diverse listing data (including photos, agent info, and rental/sales listings), web scraping is commonly used.
  • Popular portals include SUUMO, Homes.co.jp, and Rakumachi.
  • Scraping can yield thousands of listings daily with detailed attributes like price, floor plans, location, and amenities.
  • This approach requires handling dynamic web pages and legal considerations.

Benefits of scraping:

  • Real-time or near-real-time data freshness.
  • Extensive coverage and customization.
  • Useful for predictive modeling and market analysis4.

4. Official Statistical Data Portals​

  • The e-Stat portal of Japan provides official statistics datasets related to housing and land surveys, some available in JSON or CSV format.
  • These datasets are more aggregated and less granular than transaction-level data but useful for broader market analysis56.
