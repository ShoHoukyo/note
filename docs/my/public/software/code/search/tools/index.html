by sonnet4 2025-06-11:

recommend a few (less than 5) tools or commands (find? grep?) to scan .ts .js code for strings like "TODO" "FIXME" needs to:

  • be able to specify a few paths to scan
  • be able to specify additional custom search-patterns
  • auto ignore those in git .gitignore file (ripgrep does it?)
  • generate result (ideally both structured data like .json and a report in .html)
    • report better has some context (e.g. file a/b/c.ts "TODO: whole sentence") instead of only matches "TODO"

it does not need to:

  • check git history

GitHub Copilot: Here are 4 excellent tools for scanning TypeScript/JavaScript code for TODO/FIXME patterns:

1. ripgrep (rg) - Fast and Git-awareâ€‹

# Install via Homebrew
brew install ripgrep

# Basic usage
rg "TODO|FIXME" --type ts --type js

# Custom patterns with context
rg -A 2 -B 1 "TODO|FIXME|HACK|NOTE" src/ tests/ --json > todos.json
  • âœ… Respects .gitignore by default
  • âœ… Multiple file types and paths
  • âœ… JSON output available
  • âŒ No HTML report generation

2. leasot - Purpose-built for TODO scanningâ€‹

# Install
npm install -g leasot

# Scan with multiple patterns and output formats
leasot 'src/**/*.{js,ts}' 'tests/**/*.{js,ts}' \
  --reporter json > todos.json

leasot 'src/**/*.{js,ts}' 'tests/**/*.{js,ts}' \
  --reporter markdown > todos.md
  • âœ… Multiple output formats (JSON, Markdown, XML)
  • âœ… Custom patterns via config
  • âœ… Glob pattern support for paths
  • âš ï¸ Doesn't auto-respect .gitignore (but can use glob patterns)

3. ag (The Silver Searcher) - Fast with good output optionsâ€‹

# Install
brew install the_silver_searcher

# Search with context and custom patterns
ag --js --typescript "TODO|FIXME|HACK" src/ tests/ \
  --context=2 --group > todos_report.txt
  • âœ… Respects .gitignore
  • âœ… Good context display
  • âœ… Multiple file types
  • âŒ No direct JSON/HTML output

4. Custom Node.js script - Maximum flexibilityâ€‹

npm install glob ignore
const fs = require('fs');
const glob = require('glob');
const ignore = require('ignore');

const patterns = ['TODO', 'FIXME', 'HACK', 'NOTE'];
const paths = ['src/**/*.{js,ts}', 'tests/**/*.{js,ts}'];

// Read .gitignore
const gitignore = ignore().add(fs.readFileSync('.gitignore').toString());

const results = [];
paths.forEach(pattern => {
  glob.sync(pattern)
    .filter(file => !gitignore.ignores(file))
    .forEach(file => {
      const content = fs.readFileSync(file, 'utf8');
      content.split('\n').forEach((line, index) => {
        patterns.forEach(p => {
          if (line.includes(p)) {
            results.push({
              file,
              line: index + 1,
              pattern: p,
              content: line.trim(),
              context: content.split('\n').slice(Math.max(0, index-1), index+2)
            });
          }
        });
      });
    });
});

// Output JSON
fs.writeFileSync('todos.json', JSON.stringify(results, null, 2));

// Generate HTML report
const html = `
<html><body>
<h1>TODO Report</h1>
${results.map(r => `
  <div style="border:1px solid #ccc; margin:10px; padding:10px;">
    <h3>${r.file}:${r.line}</h3>
    <p><strong>${r.pattern}:</strong> ${r.content}</p>
    <pre>${r.context.join('\n')}</pre>
  </div>
`).join('')}
</body></html>`;
fs.writeFileSync('todos.html', html);

Recommendation: Start with leasot for structured TODO scanning, or ripgrep if you need maximum speed and Git integration. For custom HTML reports, the Node.js script gives you full control.

by perplexity.ai

Here are three effective tools and command-line approaches to scan .ts and .js code for strings like "TODO" and "FIXME", meeting your requirements for path specification, output formats, and contextual reporting:

1. todo-fixme-scannerâ€‹

  • Features:
    • Scans recursively for TODO and FIXME comments in JavaScript and TypeScript files.
    • Supports specifying directories to scan (--dir).
    • Outputs results in JSON, CSV, or console formats (--output json).
    • Provides context: includes the full comment, file path, and line number.
  • Limitations: Does not natively generate HTML reports, but JSON output can be easily converted to HTML with a small script or tool.
  • Usage Example:
npx todo-fixme-scanner --dir src --output json

You can specify multiple directories by running the command for each or by structuring your directory argument accordingly1.

2. grep (with find/xargs) for Custom Contextual Searchâ€‹

  • Features:
    • Use standard Unix tools to search for keywords with context.
    • Specify file extensions and multiple paths.
    • Output includes surrounding lines for context.
    • Output is plain text, but can be piped to scripts for JSON or HTML conversion.
  • Usage Example:
find path1 path2 -type f \( -name "*.js" -o -name "*.ts" \) \
  -print0 | xargs -0 grep -EnC 2 "TODO|FIXME"
- `-C 2` gives 2 lines of context above and below each match.
- Results show filename, line number, and the full comment line[^6].

3. fixme (Node.js CLI tool)â€‹

  • Features:
    • Scans for multiple annotation types including TODO and FIXME.
    • Configurable file patterns and paths.
    • Prints results to stdout with file, line, and comment context.
    • Can be used programmatically for further processing into JSON or HTML.
  • Usage Example:
npx fixme --path src --file_patterns "**/*.js,**/*.ts"
- Outputs annotated comments with file and line information, suitable for further formatting[^4].

Recommendationâ€‹

  • Use todo-fixme-scanner for structured JSON output and easy setup1.
  • Use grep/find/xargs for maximum flexibility and quick ad-hoc searches2.
  • Use fixme if you want a Node.js-based solution with easy integration into build scripts3.

For HTML reports, take the JSON or plain text output from any of these tools and use a simple script (Node.js, Python, etc.) to generate an HTML summary. This approach offers the most flexibility and control over the report format.
