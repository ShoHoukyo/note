external files​

sometimes external files are needed, and it's better to keep those files in S3 (than relying on downloading from external website) solution1: make a stack just for an S3 bucket, manually upload files to it

solution2: make a stack just for an S3 bucket, also include custom resource and lambda to download file to S3 if not exist

Trigger:
  Type: AWS::CloudFormation::CustomResource
  Properties:
    ServiceToken: !GetAtt SomeLambda.Arn
    ServiceTimeout: 900 # 15 minute: maximum Lambda duration
    # When values changes, will cause an update to this custom resource and therefore cause lambda to run
    S3Key: "filePath"
    ExternalUrl: "https://xxx"   # guess works fine when stack deployed soon, download only once

reference other stack​

https://docs.aws.amazon.com/AWSCloudFormation/latest/UserGuide/walkthrough-crossstackref.html

Parameters:
  NetworkStackName:
    Description: >-
      Name of an active CloudFormation stack that contains the networking
      resources, such as the subnet and security group, that will be used in
      this stack.
    Type: String
    MinLength: 1
    MaxLength: 255
    AllowedPattern: '^[a-zA-Z][-a-zA-Z0-9]*$'
    Default: SampleNetworkCrossStack   # note "Fn::ImportValue" does not work here


Fn::ImportValue:
  "Fn::Sub": "${NetworkStackName}-SecurityGroupID"


// when exporting better use:   "Value: !GetAtt resource.Arn"
