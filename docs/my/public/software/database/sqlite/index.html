multi process​

When SQLite tries to access a file that is locked by another process, the default behavior is to return SQLITE_BUSY.

https://www.sqlite.org/appfileformat.html SQLite automatically coordinates concurrent access to the same document from multiple threads and/or processes. Two or more applications can connect and read from the same document at the same time. Writes are serialized, but as writes normally only take milliseconds, applications simply take turns writing.
