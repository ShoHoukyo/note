Comparison of Recoll and Tracker​

Recoll and Tracker are both desktop search tools available on Linux systems, but they differ in features, indexing approach, and resource usage.

Recoll​

  • Recoll is a desktop document search application supporting full-text search of many file formats including PDFs, using the Xapian search engine library1.
  • It is known for powerful full-text indexing and search capabilities, including relevance ranking, highlighting search terms within documents, and navigation to the exact location of search terms in files like PDFs2.
  • Recoll performs indexing as a batch process and does not require running background daemons continuously, which can save system resources when not actively indexing34.
  • It can be faster in completing content indexing compared to some other tools but tends to use more CPU during indexing (up to 50% CPU usage reported)2.
  • Recoll requires installation of specific dependencies for PDF and other format support, which may require some setup effort on Debian or other distributions34.

Tracker​

  • Tracker is a metadata and content indexing service integrated with GNOME and other Linux desktops, designed to index files, documents, images, music, and more at high speed5.
  • It runs as a background daemon providing "live" indexing and search, which allows immediate availability of search results after file changes but can consume CPU continuously during indexing56.
  • Tracker uses SQLite as its backend and is designed to be lightweight in memory usage (typically 3-9 MB RAM)5.
  • It supports a broad range of file formats and metadata types but may have less comprehensive full-text search capabilities compared to Recoll, with some ambiguity historically about its full-text support34.
  • Tracker is well integrated into GNOME and related desktop environments, providing seamless search from file managers and activity views6.

Use Case Recommendations​

  • Choose Recoll if you need detailed and powerful full-text search, especially for document collections like PDFs, and prefer batch indexing without continuous background resource use.
  • Choose Tracker if you want seamless desktop integration with GNOME, continuous live indexing, and low memory footprint, and can accept somewhat less advanced full-text search features.

This comparison aligns with user discussions and reviews noting Recoll's superior full-text capabilities and Tracker's lightweight, integrated approach3452.
