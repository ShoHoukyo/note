json cmd line arg

2025-10-25 by gemini

It is common to pass a JSON string as a command-line argument, especially when you need to pass complex, structured configuration or data to a script or application. However, it's generally considered less user-friendly than using flags or a configuration file and requires careful escaping of characters to work correctly across different operating system shells.

How it Works in Deno​

In Deno, you access the raw command-line arguments and then use the built-in JSON.parse() method to convert the JSON string argument into a usable JavaScript object.

1. Accessing the Argument​

Deno exposes command-line arguments in the global Deno.args array:

  • Deno.args: A simple string[] array containing all arguments passed to your script after the script file path.

Example Command (in a Unix-like shell like Bash/Zsh):

deno run my_script.ts '{"name":"Deno","version":1.0}'

The outer single quotes ' are crucial here to ensure the entire JSON string is treated as a single argument by the shell.

2. Deno Script to Parse JSON​

In your Deno script (my_script.ts), you would read the first argument and parse it:

// my_script.ts

// 1. Get the array of arguments (everything after the script file)
const args = Deno.args;

if (args.length === 0) {
  console.log("Please provide a JSON string as an argument.");
  Deno.exit(1);
}

// The JSON string should be the first argument
const jsonString = args[0];

try {
  // 2. Use JSON.parse() to convert the string to a JavaScript object
  const data = JSON.parse(jsonString);

  console.log("Successfully parsed JSON data:");
  console.log(`Name: ${data.name}`);
  console.log(`Version: ${data.version}`);

} catch (error) {
  console.error("Error parsing JSON string:", error);
  Deno.exit(1);
}

/*
To run:
$ deno run my_script.ts '{"name":"Deno","version":1.0}'

Output:
Successfully parsed JSON data:
Name: Deno
Version: 1
*/

Note on Escaping​

The biggest challenge with passing a JSON string is shell escaping. Since JSON uses double quotes (") internally, you must enclose the entire JSON string in single quotes ('...') in most Unix-like shells (like Bash or Zsh) to prevent the shell from interpreting the internal double quotes.

On Windows, escaping rules for cmd.exe or PowerShell can differ, often requiring backslashes (\) to escape the internal double quotes.

  • Bash/Zsh: 
    deno run my_script.ts '{"key":"value with \"quotes\""}'
  • Windows cmd.exe (more complex, often requires escaping internal quotes with \"): 
    deno run my_script.ts "{\"key\":\"value with \\\"quotes\\\"\"}"

Due to these complexities, for structured data, the more idiomatic approach is often:

  1. Read the JSON from a file, passing the file path as a command-line argument (e.g., deno run my_script.ts --config ./config.json).
  2. Use a library like deno.land/std/cli/parse-args (formerly std/flags) to parse conventional flags (--key=value) and handle the structured data internally or load it from a separate source.
